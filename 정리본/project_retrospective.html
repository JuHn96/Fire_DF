회고 요약본

1. 프로젝트 개요

  • 프로젝트명: AI 기반 실시간 CCTV 화재 감지 시스템
  • 한 줄 설명: 기존 CCTV 영상을 활용해 AI로 화재를 실시간 감지하고 대시보드·카카오 알림으로 전달하는 관제용 웹 시스템
  • 수행 기간: 25.12.09-26.01.06
  • 팀 역할:
    • AI: 모델 학습·튜닝, RTSP 처리
    • BE: FastAPI API, 이벤트 로직, WebSocket, DB 연동
    • DB: 스키마 설계, 정규화
    • FE: Vue 대시보드, 로그/통계, CCTV 관리 화면

2. 프로젝트 목적

기술적 목적

  • 기존 CCTV 인프라 그대로 사용하면서 AI 기반 화재 감지 기능 추가
  • 실시간 대시보드 + 카카오 알림으로 빠른 대응 지원
  • Docker 기반으로 개발/테스트/운영 환경을 통합

학습/역량 목적

  • AI / FE / BE / DB 전체 파트를 하나의 실제 서비스로 연결
  • 팀 단위 협업 및 문서화 경험 확보
  • WebSocket, RTSP, 카카오 등 실무 기술스택 경험

3. 시스템 구성 요약

  • 입력: CCTV/NVR RTSP 스트림
  • AI: 화재 모델, 신뢰도/시간 기준 이벤트 발생
  • BE: 이벤트 로직, WebSocket, 카카오, DB 연동
  • DB: 이벤트 로그, CCTV 정보, 템플릿 등 저장
  • FE(VUE): 대시보드, 로그/통계, CCTV 관리, 마이페이지
  • 인프라: Docker Compose 통합 실행 구조

4. 수행 과정 요약

  • 주제 선정 → 전체 구조 확정(“CCTV + AI + 웹 + 카카오”)
  • 역할 분담 후 깃허브 협업 구조 세팅
  • DB 스키마·API·WebSocket·카카오·이벤트 플로우 순으로 백본 구축
  • 대시보드·로그/통계·CCTV 관리 등 UI 구조 정리
  • 기업프로젝트 제출에 맞춘 문서/플로우/아키텍처 전체 구조화

5. 성과·잘된 점

  • “CCTV → AI → BE → DB → FE → 카카오” 전체 흐름을 하나의 서비스로 연결
  • 역할 간 책임이 분명해 협업 효율적
  • 문서화 병행으로 전체 구조를 팀원 전원이 동일하게 이해
  • RTSP, Docker, WebSocket, 카카오 등 실무 핵심 기술을 실제 적용

6. 팀 협업 회고

  • 백엔드·AI·FE·DB가 서로의 관점에서 요구사항을 반복적으로 조율하며 협업 경험을 쌓음
  • API 정의가 단순한 스펙이 아니라 팀 간 약속이라는 점을 체감함
  • 문서화가 쌓일수록 프로젝트 의사결정이 명확해지고 작업 효율이 올라감
  • 역할 분담이 되어 있어도 한 서비스 안에서 생기는 문제를 함께 풀어가는 과정이 의미 있었음

7. 향후 개선 방향

  • 화재 + 인명 감지 모델(쓰러짐/움직임 없음 등) 추가
  • ELK·Prometheus/Grafana 등을 이용한 고급 로그·모니터링 시스템 구축
  • CI/CD 자동화(GitHub Actions)로 빌드/배포 파이프라인 구성
  • 네트워크 장애, 카메라 오류, 저장소 임계점 등 실제 운영 이슈 대응 로직 강화
