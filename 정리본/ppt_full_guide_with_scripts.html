🔥 최종 보고서 / 발표(PPT) 구성 – 디테일 확장 버전

1. 표지 (Title Slide)

슬라이드 요소

  • 프로젝트명: 지능형 CCTV 화재 감지 시스템
  • 부제: AI 기반 실시간 모니터링 + 자동 알림 시스템
  • 팀명 / 팀원 역할
  • 발표자 이름
  • 제출일자 / 발표일

발표 스크립트

“본 프로젝트는 기존 CCTV 환경 위에서 동작하는 AI 기반 화재 감지 및 실시간 알림 시스템입니다. 화재를 조기에 탐지하고 관리자에게 즉시 전달하여 피해를 최소화하는 것을 목표로 합니다.”

2. 프로젝트 개요

슬라이드 내용

  • 실내외 CCTV는 많지만 대부분 “녹화용”일 뿐
  • 관제실 인력은 한 번에 많은 화면을 감시해야 함
  • 화재의 ‘초기 30초’ 대응이 피해를 크게 좌우
  • AI가 사람이 놓칠 수 있는 초기 신호를 자동으로 탐지

발표 스크립트

“화재는 빠른 초기 대응이 가장 중요하지만, 실제 관제 환경은 사람이 모든 화면을 지속적으로 감시하기 어렵습니다. 따라서 AI가 실시간으로 CCTV 속 상황을 분석해 초기 화재 징후를 즉시 포착하는 구조가 필요했습니다.”

3. 문제 정의 (Current Problems)

슬라이드 내용

  • 🔴 기존 CCTV 시스템의 한계
    • 단순 녹화 → 실시간 이상 탐지는 불가능
    • 초기 연기/불꽃을 사람이 놓침
    • 관리자 부재 시 대응 지연
  • 🔴 경보 시스템과 CCTV가 서로 독립되어 있음

발표 스크립트

“기존 CCTV는 단순 모니터링 장비라, 실제로 불꽃이나 연기가 발생했을 때도 즉시 탐지하기가 어렵습니다. 또한 알림 시스템은 CCTV와 분리되어 있어 정보 전달이 늦어지는 문제가 있습니다.”

4. 프로젝트 목표

슬라이드 내용

  • 실시간 영상 기반 화재 감지
  • WebSocket 기반 대시보드에 즉시 표출
  • 카카오톡 메시지 알림 자동 전송
  • 이벤트 로그 기록 & 히스토리 조회
  • 간단한 인프라로 어디서든 쉽게 설치 가능

발표 스크립트

“프로젝트의 핵심 목표는 화재를 빠르고 정확하게 감지하고, 이를 관리자에게 즉시 전달하여 대응 속도를 극대화하는 것입니다.”

5. 전체 시스템 구성도 (Architecture)

슬라이드 구성 요소

  • CCTV (RTSP 스트림)
  • AI 서버 (YOLO 기반 화재 감지)
  • Backend (FastAPI)
  • MySQL
  • WebSocket 서버
  • Frontend (Vue + Nginx)
  • 카카오 알림 모듈

데이터 흐름 구조

CCTV → AI → Backend → DB → FE(WebSocket 실시간 반영) → 관리자
Backend → 카카오 API

발표 스크립트

“CCTV로 들어오는 영상은 AI 서버에서 실시간 분석되고, 화재가 감지되면 백엔드로 이벤트가 전달됩니다. 백엔드는 이를 DB에 저장하고, 동시에 WebSocket을 통해 프론트에 즉시 푸시합니다.”

6. 데이터 흐름도 (Detailed Flowchart)

흐름 단계

  • RTSP 영상 수신
  • AI 모델 추론
  • 화재/연기 판단
  • Backend 내부 이벤트 처리
  • DB 저장
  • WebSocket 실시간 알림
  • Frontend UI 갱신
  • 카카오톡 발송

발표 스크립트

“AI는 프레임마다 화재 여부를 판단합니다. 판단된 이벤트는 백엔드로 전달되며, 백엔드는 DB 저장, WebSocket 알림, 카카오톡 메시지 발송을 동시에 처리합니다.”

7. 기능 상세 설명

7-1. 실시간 CCTV 모니터링

  • AI가 화재 가능성을 박스로 표시
  • 여러 카메라를 동시에 표시
  • 감지 레벨을 단계별(관심/주의/경고)로 표시

7-2. 이벤트 로그 화면

  • 시간별 이벤트 히스토리 확인
  • 감지 시점의 이미지 캡처 저장
  • 정렬 및 필터링 기능 제공
  • 이벤트 상세 정보 조회

7-3. 카카오 알림 기능

  • 사용자 등록 템플릿 관리
  • 레벨별로 메시지 내용을 다르게 전송
  • 전송 성공/실패 기록 저장

7-4. 관리자 기능

  • CCTV 목록 등록/수정/삭제
  • 카카오 템플릿 값 및 설정 관리
  • 시스템 공통 설정

발표 스크립트

“관리자는 대시보드를 통해 모든 CCTV 화면을 한눈에 볼 수 있습니다. 화재가 발생하면 즉시 알림창이 뜨며, 카카오톡으로도 알림이 전달됩니다.”

8. 역할 분담 (Team Roles)

AI

  • YOLO 기반 화재 모델 훈련
  • RTSP 영상 처리 파이프라인 개발
  • Backend 이벤트 전송 모듈 구현

Backend

  • FastAPI 기반 REST API 개발
  • 이벤트 로직 및 단계별 처리
  • DB 저장 및 조회
  • WebSocket 서버 구현
  • 카카오 메시지 전송 모듈 구현

Frontend

  • Vue 기반 관리자 UI 개발
  • 실시간 이벤트 렌더링
  • CCTV 모니터링 화면 구성
  • 통계/로그 화면 구현

DB

  • 스키마 설계
  • 정규화 진행
  • 인덱싱 및 성능 튜닝

발표 스크립트

“각 파트가 서로 분리되어 있으면서도 긴밀하게 연동된 구조로 설계했습니다. 특히 BE–AI–FE 간 실시간 연결이 핵심이었습니다.”

9. 핵심 기술 설명

사용 기술

  • YOLO 모델(불꽃/연기 감지)
  • FastAPI REST + WebSocket
  • MySQL (InnoDB)
  • Vue3 + Vite + Nginx
  • Docker Compose (컨테이너 통합 실행)

발표 스크립트

“전체 시스템은 Docker Compose 하나로 전체 서비스가 자동으로 연결됩니다. AI–Backend–Frontend가 fire_net 네트워크에서 서비스 이름으로 서로 통신합니다。”

10. 시연(데모) 설명

시연 순서

  • CCTV 스트림 입력
  • AI 감지 (Bounding Box 표시)
  • Backend 이벤트 수신
  • WebSocket 알림으로 FE에 반영
  • 대시보드 팝업 표시
  • 카카오톡 알림 도착
  • 로그 화면에서 히스토리 확인

발표 스크립트

“이 시연에서는 실제 CCTV와 유사한 RTSP 영상으로 화재를 감지하고, 알림이 어떻게 전달되는지 실시간으로 보여드립니다.”

11. Docker 기반 실행 구조 설명

핵심 원리

  • 하나의 명령: docker compose up -d --build
  • fire_net 내부에서 서비스 이름으로 서로 연결
  • MySQL 볼륨 유지
  • frontend만 8080으로 외부에 노출

발표 스크립트

“실행 환경이 달라도 컨테이너 단위로 동일하게 실행되며, AI–백엔드–DB–프론트 전체가 자동으로 기동합니다.”

12. 결과 및 성과

성과

  • 실시간 감지 및 알림 기능 완성
  • 전체 시스템을 Docker 기반으로 실행
  • WebSocket 실시간 구조 구현
  • 실제 서비스 가능한 수준의 프로토타입 완성

발표 스크립트

“이번 프로젝트는 실사용 가능한 수준의 프로토타입을 구축했다는 점에서 의미가 큽니다.”

13. 향후 개선 방향

예정된 확장 기능

  • 인명 구조 감지(쓰러짐 감지 등) 추가
  • GPU 서버 최적화 → 지연시간 감소
  • ELK Stack 기반 로그 분석
  • 다중 카메라 대규모 지원

발표 스크립트

“AI 모델을 더 확장해 인명 구조까지 넓히고, 로그 분석 시스템을 강화하여 기업 환경에도 바로 적용 가능한 구조로 만들 계획입니다.”

14. 마무리 및 Q&A

발표자 마무리 멘트

“이상으로 지능형 CCTV 화재 감지 시스템 발표를 마치겠습니다. 질문 있으시면 편하게 말씀해주세요.”
